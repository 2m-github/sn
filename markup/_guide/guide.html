1. 언어코드

각 언어별로 폴더를 구분

  • 한국어 : /ko_KR/
  • 영어 : /en_US/
  • 중국어 : /zh_CN/
  • 베트남어 : /vi_VN/
  • 몽골어 : /mn_MN/
  • 우즈베크어 : /uz_UZ/
  • 태국어 : /th_TH/
  • 캄보디아(크메르어) : /km_KH/
  • 네팔어 : /ne_NP/
  • 미얀마(버마어) : /my_MM/

2. 폴더구조

  • HTML : /언어코드/*.html
  • CSS : /언어코드/css/*.css
  • JS : /언어코드/js/*.js
  • image : /언어코드/img/*.jpg|png|gif

3. HTML

charset="UTF-8"

상대경로 사용

각 언어별 html태그에 Language Attribute를 반드시 표기

    < html lang="en-US">

4. CSS

charset="UTF-8"

상대경로 사용

html 기본 폰트 사이즈 - font-size:10px

단위는 %로 적용하는 요소 외에 rem만 사용 (line-height에 사용되는 em 제외)

1. HTML

html의 lang 어트리뷰트와 강제 줄바꿈을 위한 br태그를 제외하고 10개국어의 html 태그와 사용된 클래스는 모두 동일


    < html lang="ko-KR"> 
    < html lang="en-US"> 
    < html lang="zh-CN"> 
    < html lang="vi-VN"> 
    < html lang="mn_MN"> 
    < html lang="uz-UZ"> 
    < html lang="th-TH"> 
    < html lang="km-KH"> 
    < html lang="ne-NP"> 
    < html lang="my-MM">

향후 추가될 언어에서 번역본 적용 후 틀어지는 레이아웃을 CSS만으로 처리가 불가능할 경우 부득이 html 수정을 최소화하는 방법(ex. 더블클래스 적용)으로 작업 후 개발측에 분기처리 요청

2. CSS

CSS는 언어별로 조금씩 상이하며, 새로운 언어가 추가될 경우 비슷한 길이의 문자열을 가진 언어의 CSS를 기본 베이스로 하고, 세세한 수정이 필요한 경우 커스터마이징 함.
